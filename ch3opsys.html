Operating System Terminology

TERM DEFINITION
Operating System The software that controls the overall operation of a computer, including file management, the user interface, and the program execution environment.
Job Queue A storage space which controls the order of the execution of programs in the same order in which they arrive. This is also called "first in, first out" or FIFO.
FIFO "First in, first out." That is, programs are executed and removed from the queue in the same order that they arrived in.
Interactive Processing Allows a program to receive instructions from the user while it is already being executed.
Time-sharing A system which can serve multiple users by allowing multiple programs to be executed simultaneously(-ish) on one common computer.
Multiprocessing The splitting-up of a computer's time into short intervals. If several programs each are given one interval, the computer can switch between them, thus executing many programs at once.
Application Software Programs which are specialized to performing tasks particular to that machine.
System Software Programs which are common to computer systems in general.
Utility Software Programs for performing activities that are fundamental to computer installations but not included in the operating system, but work to customize it.
Graphical User Interface (GUI) Displays files and programs as images on a display which can be manipulated visually, such as by mouse.
Shell An older style of user interface, in which all information is displayed as text.
Kernel The internal part of an operating system. It contains the software for basic computer functions.
Device Driver Software units which communicate with controllers and attached devices to carry out operations on those external machines.
File Manager Coordinates the use of the computer's files. It holds information about where each file is and who is allowed to access it.
Memory Manager Coordinates the use of the computer's main memory, which controls which and when programs are executed, and how much memory is assigned to each.
Virtual Memory The swapping of a program between main memory and mass storage, giving the illusion that a computer has more memory space than it really has.
Boot-Strapping The procedure performed when a computer turns on, which transfers a computer's operating system from mass storage into main memory.
ROM "Read-Only Memory." As the name suggests, this is computer memory whose contents can be read, but not altered.
Deadlock The condition where multiple processes are blocked from progressing because each is waiting for a resource that has been allocated to another program.
Spooling A technique for avoiding deadlock where the computer takes note of program requests in mass storage, rather than flooding the resource with multiple conflicting requests at once.
Super User (or Administrator) User account with highly-privileged access to a computer's operating system, altering settings like user permissions, and monitoring user activity.
Multi-Factor Authentication The requirement for a user to provide multiple sources of evidence to prove that they own an account.
