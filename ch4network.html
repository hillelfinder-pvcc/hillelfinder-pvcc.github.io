Network Terminology

TERM DEFINITION
Network Classifications
  1. LAN
  2. MAN
  3. WAN
 Network Classifications
  1. "Local Area Network:" A collection of computers in a single building or complex.
  2. "Metropolitan Area Network:" A network of greater size, which spans a community.
  3. "Wide Area Network:" A network of even greater size, spanning multiple cities or continents.
Network Models
  1. Bus Network
  2. Star Network
 Network Models
  1. Multiple machines are all connected to a common line of communication.
  2. Multiple machines are all connected to a common focal machine.
Protocols
  1. CSMA/CD Ethernet
  2. CSMA/CA Wifi
 Protocols
  1. "Carrier Sense, Multiple Access with Collision Detection:" A bus protocol which dictates that all messages be broadcast to all machines on the bus. Every machine listens for signals sent to it, and only sends signals when the bus is quiet.
  2. "Carrier Sense, Multiple Access with Collision Avoidance:" A protocol similar to CSMA/CD, but that gives message-sending priority to machines which have been waiting for longer for a longer period of time.
Combining Networks
  1. Repeater
  2. Bridge
  3. Switch
  4. Router
  5. Gateway
 Combining Networks
  1. A device which passes and amplifies signals between buses.
  2. Similar to a repeater, but hesitates to forward messages until it knows that the message is for a computer on the other side of the connection.
  3. A bridge which connects to more than two machines.
  4. A special-purpose computer which links multiple networks of machines together into an "internet," while allowing each network to maintain its unique structure.
  5. The point at which a network is linked to an internet; literally the gate between multiple networks.
Client/server Model
  1. P2P: Peer-to-Peer
  2. Distributed System
  3. Cloud Computing
 Client/server Model
  1. A method of network communication where multiple machines ask for and execute processes for each-other, rather than a central source.
  2. Software units that execute as processes on multiple computers at once.
  3. A type of distributed system, in which a process is sent out to a large number of computers all sharing a network.
Internet Addressing
  1. Tier 1
  2. Tier 2
  3. Tier 3
 Internet Addressing
  1. The highest-speed and highest-capacity of WANs which are the foundation of the Internet.
  2. More regional networks which connect to Tier 1 ISPs.
  3. An "independent internet." A single authority which supplies Internet access to individual homes and businesses.
IP Address
  1. ICANN
  2. DNS
  3. TLD: Top-level Domain
  4. Country-code TLDs
 IP Address
  1. The "Internet Corporation for Assigned Names and Numbers." A non-profit which awards IP addresses to ISPs.
  2. "Domain Name System." A network of name servers, which converts machine addresses between mnemonic domains, which are easy for humans, and IP address, which is required for computers.
  3. Special domain classifications for organizations, such as ".edu" ".gov" and ".org"
  4. Special domain classifications for countries, such as ".au" and ".ca"
Internet Protocols
  1. FTP
  2. HTTP
  3. SMTP
 Internet Protocols
  1. "File Transfer Protocol:" A simple application for listing and copying files across a network.
  2. "Hyper-Text Transfer Protocol:" A powerful application for webpages and browser using the Internet.
  3. "Simple Mail Transfer Protocol:" An application for two computers in a network to send email messages between them.
World Wide Web
  1. URL
  2. Client-side Activities
  3. Server-side Activities
 World Wide Web
  1. "Uniform Resource Locator:" A unique address given to each document on the Web.
  2. Activities on a webpage performed by the client, such as a browser.
  3. Activities on a webpage performed by a server, such as a webserver.
Internet Four-Layer Software
  1. Application Layer
  2. Transport Layer
  3. Network Layer
  4. Link Layer
 Internet Four-Layer Software
  1. Consists of those software units (such as clients and servers) that use Internet communication to carry out their task, such as translating between domain names and IPs.
  2. Accepts messages from the application layer and ensures that they are properly formatted for transmission over the internet, sent as discrete packets of data.
  3. Decides which "direction" a packet of information should be sent over the Internet.
  4. In charge of receiving and transmitting packets through the local network and into the Internet.
Internet is:
  1. Packet-switching Network
  2. Fault Tolerant
  3. Redundant
 Internet is:
  1. Manages and tolerates the many packets which are constantly being sent by machines all over the world.
  2. By checking how many times a packet has been sent, it can stop packets from endlessly cycling through the Internet.
  3. When a channel is too congested, an alternate pathway for packets can be found, reducing delays.
Cybersecurity
  1. Malware
  2. Virus
  3. Worm
  4. Trojan Horse
  5. Spyware
  6. Phishing
  7. DDos Attack
 Cybersecurity
  1. General term for malicious software.
  2. Software which "infects" a computer by inserting itself into programs that already exist on the machine.
  3. An autonomous program that transfers itself through a network, taking up residence in computers and forwarding copies of itself to other computers.
  4. A program that enters a computer system disguised as a desirable program.
  5. Software that collects information about activities at the computer on which is residues and reports that information back to the instigator of the attack.
  6. A technique of obtaining information explicitly by asking for it from unwitting users, usually by posing as a trusted source.
  7. "Distributed Denial of Service" Attak: A technique overloading a target computer/server with messages from multiple machines sharing a network.
Protection
  1. Firewall
  2. Spam Filter
  3. Proxy Server
  4. Antivirus Software
  5. Cryptography
  6. Public Key Encryption
 Protection
  1. A primary prevention technique to filter traffic passing through a point in a network/computer by filtering and/or blocking messages.
  2. A special type of firewall designed to block unwanted emails.
  3. A software unit which acts as an intermediary between a client and a server, with the goal of shielding the client from any adverse action by the server.
  4. Software which is used to detect and remove different types of malware.
  5. The study of sending and receiving messages securely in the presence of adversaries.
  6. A system of encryption in which the encryption and decryption algorithms are known, but only one of two "keys" (passwords) are known by any user.
