Chapter 6: Programming Languages

    Machine language
    • An encoding of a sequence of instructions as numerical digits
    Debugging
    • The task of locating and correcting errors in a machine language
    Assembler
    • A system which converts mnemonic expressions into machine language instructions
    Assembly language
    • A mnemonic system for representing programs
    Translator
    • A program which translates high-level primitives into machine-language programs
    Compiler
    • Compiles several machine instructions into short sequences to simulate the activity requested by a single high-level primitive
    Interpreter
    • A program similar to a translator except that they execute instructions as they were translated instead of recording the translated version for future use
    Cross-platform software
    • A term used to describe a program which works independently of the design of computer’s operating system and hardware
    Natural language
    • Languages (such as English, german, latin, etc) which have evolved over time
    Formal language
    • Languages (such as programming languages) which have extremely precise grammars
    Machine independence
    • A set of data primitives (language) which does not rely on the characteristics of any particular machine
    High-level programming language
    • A programming language with strong abstraction from the details of the computer
    Object-oriented programming (OOP)
    • A paradigm in which a software system is viewed as a collection of units, called objects, each of which is capable of performing the actions that are immediately related to itself as well as requesting actions if integer objects
    Scripting language
    • A programming language that employs a high-level construct to interpret and execute one command at a time
    Data types:
    • Integer
      • Numerical data consisting of whole numbers (stored in the computer using two’s complement notation)
    • Float
      • Numerical data that might contain values other than whole numbers (stored in floating-point notation)
    • String
      • Data represented as text in ASCII, rather than numerically
    Primitive data type
    • Types of data such a "int" for integers and "float" for fractional numbers
    Data structure
    • The conceptual shape or arrangement of data
    Python collection data structures
    • List
      • A collection of data whose entries are arranged sequentially. List items are ordered, changeable, and allow duplicate values
    • Tuple
      • A collection of data whose entries are arranged into rows of a table, which is ordered and unchangeable
    • Set
      • A collection which is unordered, unchangeable, and unindexed
    • Dictionary
      • A collection which is ordered, changeable and do not allow duplicates
    Function
    • A set of instructions for performing a task that can be used as an abstract tool by pothere program units
    Local variable
    • A variable declared within a function
    Global variables
    • A variable whose scope is not restricted to a particular part of a program
    Source program
    • A program in its original form before it is converted into another language
    Object program
    • A program which has already undergone translation from another language
